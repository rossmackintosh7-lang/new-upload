Retail setup

How to connect Stripe for a retail website

Use this guide before taking live product payments through a PBI retail template.

Customer steps

  1. Open your PBI project in the builder.
  2. Go to Retail shop and click Connect Stripe.
  3. Stripe opens a secure hosted onboarding form.
  4. Create or log in to Stripe and complete the requested business verification.
  5. Return to PBI. PBI automatically stores the connected account, so you do not need to copy an acct_... ID.
  6. Add up to 10 products in PBI with names, prices, stock and images.
  7. Choose whether Stripe Tax should be on.
  8. Add your shipping label and amount.
  9. Save, publish, and run a small test order.

Important payment note

For live retail websites where the business receives the money directly, PBI uses Stripe Connect. The connected account must be properly onboarded and connected to the platform before customer payments are taken.

If Stripe is not connected, PBI blocks retail checkout and asks the business to complete Stripe onboarding.

For a very simple shop, Stripe Payment Links can still be used as a fallback, but Payment Links are not a full stock/cart/order system inside PBI.

Admin checklist

  • Set STRIPE_SECRET_KEY as a Cloudflare Secret.
  • Set STRIPE_WEBHOOK_SECRET as a Cloudflare Secret.
  • PBI_RETAIL_CONNECT_ENABLED=true is now enabled by default in the launch build. Keep STRIPE_SECRET_KEY and STRIPE_WEBHOOK_SECRET as Cloudflare Secrets.
  • For testing only, set PBI_RETAIL_ALLOW_PLATFORM_PAYMENTS=true if payments should go into the platform Stripe account.
  • Run the D1 schema after upload.
  • Test checkout, webhook, order dashboard and stock reduction before public selling.
