PBI BUILDER REAL LINKED UPLOAD/SAVE FIX

Straight answer:
- Before this patch, the builder was only partially linked.
- The template editor could change a preview, but customer save/publish was not guaranteed because the save route was admin-only.
- The advanced canvas was not reliably writing back to the same section data.
- Image upload was missing from the section inspector.

This build fixes:
- Added customer-safe /api/builder/project-sections for section load/save
- Added image upload endpoint with R2 support and data URL fallback
- Added /api/builder/publish-sections to snapshot saved sections when publishing
- Added published_project_sections to combined D1 schema
- Replaced pbi-builder-sections.js with linked editor, upload support, live preview update, save and publish hooks
- Linked advanced canvas save/apply to the same project_sections data
- Added linked builder/upload CSS

New routes:
- /api/builder/project-sections
- /api/builder/upload-image
- /api/builder/publish-sections

Image upload:
- Customers can upload an image in the selected section inspector.
- If Cloudflare R2 binding PBI_ASSETS exists, files save there.
- Without R2, images work as data URLs for testing.

Production storage recommendation:
- Add an R2 bucket binding named PBI_ASSETS.
- Optional variable: PBI_ASSETS_PUBLIC_URL.

D1 schema needed:

CREATE TABLE IF NOT EXISTS published_project_sections (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  section_order INTEGER DEFAULT 0,
  section_type TEXT NOT NULL,
  title TEXT,
  text TEXT,
  button TEXT,
  image TEXT,
  layout TEXT DEFAULT 'standard',
  background TEXT DEFAULT '#fff8f1',
  accent TEXT DEFAULT '#bf5c29',
  padding TEXT DEFAULT 'comfortable',
  align TEXT DEFAULT 'left',
  hidden INTEGER DEFAULT 0,
  body_json TEXT,
  published_at TEXT DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX IF NOT EXISTS idx_published_project_sections_project_id ON published_project_sections(project_id);


Test:
1. Open /builder/?project=YOUR_PROJECT_ID
2. Change a section title: the Website window should update immediately.
3. Upload an image: the image should appear in the Website window.
4. Click Save sections or Save project.
5. Refresh the builder: changes should reload from D1.
6. Click Publish website: sections snapshot to published_project_sections.
