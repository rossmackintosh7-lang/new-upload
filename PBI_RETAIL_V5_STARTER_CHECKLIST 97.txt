PBI RETAIL V5 STARTER UPGRADE

Added:
- Retail shop editor in the AI Builder.
- Up to 10 products per website.
- Product fields: name, description, price, stock, SKU, image, Stripe Price ID, Payment Link fallback, visible/hidden, stock tracking.
- Retail settings: enable shop, currency, Stripe connected account ID, Stripe Tax on/off, shipping label and shipping amount.
- Published retail site product grid and customer basket.
- Stripe Checkout endpoint: /api/retail/create-checkout
- Retail orders endpoint: /api/retail/orders
- Retail order dashboard: /retail-orders/
- Stripe webhook handling for retail checkout.session.completed events.
- Stock reduction after paid webhook.
- D1 tables: retail_orders and retail_order_events.
- Retail Stripe setup guide: /help/retail-stripe-setup/
- Shop template now includes sample products.

Important Stripe setup:
- For real customer/seller payments, configure Stripe Connect and set PBI_RETAIL_CONNECT_ENABLED=true.
- Each shop must have a connected Stripe account ID saved in the builder, starting with acct_.
- If Connect is not enabled, retail checkout is blocked by design.
- For platform-only testing, set PBI_RETAIL_ALLOW_PLATFORM_PAYMENTS=true. Do not use that for real customer shops unless payments are intentionally going into the platform account.

After upload, run:
npx wrangler d1 execute d1-template-database --file=./database/pbi-combined-schema.sql --remote

Test flow:
1. Choose Shop / Retail template.
2. Create account or open a retail project.
3. Builder -> Retail shop -> add products.
4. Set stock and prices.
5. Save project.
6. Publish project.
7. Open live /site/{slug}/ page.
8. Add product to basket.
9. Try checkout.
10. Confirm it blocks if Connect/Stripe is not configured.
11. Configure Stripe and retest.
12. Check /retail-orders/ after successful webhook.
